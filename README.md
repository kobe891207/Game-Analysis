# 探討神魔之塔保底抽數設計
## 專案簡介
本專案以手機遊戲《神魔之塔》之抽卡系統為研究對象，分析其抽卡機率、保底機制與課金價格設計之間的關聯性。
透過：
* 機率推導
* Monte Carlo Simulation（蒙地卡羅模擬）驗證
* 百分位數分析
* 課金價格分析

探討遊戲如何透過抽卡機制與商城定價影響玩家行為與課金決策。
## 研究動機
現代手機遊戲普遍採用：
* 抽卡系統
* 保底機制
* 階梯式課金定價

來提升玩家參與度與營收。

然而，這些機制背後通常並非獨立設計，而是存在高度關係。

本研究希望透過數據分析與模擬方法，探討：
1. 玩家平均需要多少抽才能取得目標角色
2. 保底機制如何影響玩家體感
3. 商城石頭價格是否與保底機制存在對應關係
4. 遊戲公司如何透過價格設計提升玩家黏著度
## 研究方法
### 1. 機率模型
假設：

* 每次抽卡獲得目標角色機率為 p
* 玩家每次抽卡互相獨立

透過機率公式
1. 玩家抽卡次數期望值
``` python
def exp(p, pity): #計算抽大獎的期望抽數
    e = 0

    if pity is not None:
        for i in range(1, pity):
            e = e + i*p*pow(1-p, i-1)
            
        e = e + pity*1*pow(1-p, pity-1)
        return e
    else:
        e = 1/p
        return e
```
2. 抽卡次數超過期望值的機率密度函數
``` python
def PDF(p, exp): #計算機率密度函數
    pdf = 0

    for i in range(math.ceil(exp)):
        pdf = pdf + p*pow(1-p, i-1)

    return 1 - pdf
```
### 2. 蒙地卡羅驗證
使用 Python 模擬大量玩家抽卡行為

模擬流程：
每位玩家持續抽卡直到取得目標角色
記錄玩家消耗抽數
重複模擬 1,000,000 次
統計抽數分布與百分位數
```python
def simulate_one_player(p = 0.01, pity = 35):
    """
    模擬一位玩家抽到目標卡需要幾抽
    p: 每抽中目標卡的機率
    pity: 保底抽數，若沒有保底可設為 None
    """
    count = 0

    while True:
        count += 1

        # 如果有保底，抽到 pity 次就直接中
        if pity is not None and count == pity:
            return count

        # 隨機抽卡
        if random.random() < p:
            return count
```
``` python
def monte_carlo_simulation(num_players=1000000, p = 0.01, pity = 35):
    results = []

    for _ in range(num_players):
        draws = simulate_one_player(p, pity)
        results.append(draws)

    return np.array(results)
```

## 結果觀察與討論
### 1. 無保底機制下，玩家成本不確定性較高
在無保底卡匣中，即使理論期望值可以計算出平均抽數，我們發現大約1/3的玩家抽數大於期望值，並且有10%以上的玩家抽數遠高於期望的情況。

例如：

1. 合作卡匣中獎機率 1%，無保底：期望100抽，有10%以上玩家必須花200抽以上
2. 黑金卡匣中獎機率 0.8%，無保底：期望125抽，有10%以上玩家必須花250抽以上

這類卡匣雖然可以用期望值估計平均成本，但實際玩家體驗可能差異很大。

部分玩家可能少量抽數即中獎，也可能有玩家需要非常大量抽數才取得目標角色，容易造成玩家的反彈

### 2. 保底機制能有效限制最大成本
加入保底機制後，玩家的最大抽數會被限制。

例如：

1. 合作卡匣 35 抽保底：期望值從100抽降為30抽
2. 黑金卡匣 40 抽保底：期望值從125抽降為35抽

保底機制可以降低極端情況發生，使玩家能夠預期最壞情況下的資源消耗。

這對玩家心理有明顯影響，因為玩家不再需要承擔無上限的抽卡風險，並且在加倍期間，超過一半的玩家可以少於原本期望值，更大幅增加抽卡意願。

### 3. 保底成本與課金方案存在高度關聯
根據結果顯示，保底機制下玩家平均抽數約落在 30 抽附近。
由於《神魔之塔》中一次抽卡通常需要 5 顆魔法石，因此玩家平均成本為150顆魔法石

而遊戲商城中最高階課金方案提供的石頭量也剛好為 150 顆。

這代表遊戲可能將：一次主要課金金額 ≈ 一次完整保底循環

讓玩家能在花費一單的價格下，得到良好遊戲體驗，製造雙贏局面。

### 4. 玩家行為分析
從上述我們可以知道平均魔法石花費約等於課金一單價錢，但結果顯示其實大部分玩家抽數為保底，還需25~50顆魔法石彌補，因此官方想出以下獲得魔法石方法：

* 每日登入
* 完成日常任務
* 參與活動關卡
* 開放黑金卡匣伴隨儲值回饋

因此，保底機制不僅影響課金行為，也可能提高玩家的每日活躍度與遊戲黏著度。

## 結論
本專案透過機率分析與蒙地卡羅模擬發現，抽卡保底機制能有效降低玩家成本的不確定性，並限制極端抽不到目標角色的情況。

此外，保底所需石頭量與商城高階課金方案之間存在高度對應關係。

因此，本研究認為《神魔之塔》的抽卡系統並非只有單純的機率設計，而是結合了：

* 機率控制
* 保底機制
* 玩家心理
* 課金誘因
* 遊戲黏著度

所形成的一套完整商業設計。
